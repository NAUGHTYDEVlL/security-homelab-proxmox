# Security Homelab – Proxmox Virtual Environment

## Overview

This project documents the design and operation of a personal cybersecurity homelab built using virtualization.  
The lab simulates a simplified enterprise security monitoring environment used to practice threat detection, log analysis, attack investigation, and system hardening.

The environment is built using Proxmox VE and consists of multiple virtual machines that replicate common infrastructure components found in real-world environments.

The homelab currently includes three virtual machines:

| Virtual Machine | Purpose |
|----------------|--------|
| ubuntu-web | Linux web server running Apache |
| siem-wazuh | Security monitoring platform |
| wireguard | Secure VPN access to the lab environment |

This architecture allows simulation of a monitored web service with centralized log collection and secure remote access.

## Security Monitoring

Logs generated by the Apache web server are ingested into the Wazuh SIEM for centralized monitoring and analysis.

This setup allows investigation of:

- Suspicious HTTP requests
- Automated vulnerability scanning
- Potential attack attempts against the web server

The monitoring environment provides hands-on experience analyzing real log data and identifying potential security events.

---

## Attacks Observed

While monitoring the environment, multiple automated attack attempts originating from international IP addresses were detected.

Observed attack activity included:

- SQL injection attempts
- Directory traversal attempts
- Automated vulnerability scanning by bots

Analysis of Apache access logs showed that these requests resulted in HTTP `404` responses, indicating that the attack attempts were unsuccessful.

---

## Defensive Measures Implemented

To reduce the attack surface of the environment and improve security, several hardening measures were implemented:

- Removal of unused services
- Disabling unnecessary system assets
- Implementation of system hardening practices

These measures help minimize potential vulnerabilities and reduce the likelihood of successful exploitation attempts.
